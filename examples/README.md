# Examples

- `portfolio.sample.json`: portfolio input for Judge runs
- `events.sample.json`: example events payload consumed by `libra-agent`
- `normalized-documents.sample.json`: example normalized document payload consumed by `libra-agent`
- `agent-responses/`: sanitized sample outputs that each sub-agent owner should match

These files are input fixtures for `libra-agent`. They mirror upstream knowledge payloads, but they are not collector implementations or source configs.
