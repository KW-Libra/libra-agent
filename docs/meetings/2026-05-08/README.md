## 발표 순서

1. PPT 1~3장으로 제목과 문제정의를 먼저 정리한다.
2. PPT 4장 요구사항 명세를 짧게 보여준다.
3. PPT 5~7장으로 기능 아키텍처와 멀티 에이전트 구조를 설명한다.
4. PPT 8~9장으로 현재 구현된 실제 흐름과 데모 경로를 보여준다.
5. PPT 10장으로 검증 계획을 말한다.
6. PPT 11장으로 팀원 repo 통합 계획을 말한다.
7. PPT 12장으로 다음 작업과 피드백 요청을 정리한다.

## 데모에서 반드시 보여줄 화면

1. 로그인 또는 대시보드 진입.
2. 목표 포트폴리오 설정 또는 현재 정의 확인.
3. Judge Run 실행.
4. History에서 `REBALANCE` 판단 확인.
5. Decision Trace 확인.
6. `KIS 현재가로 수량 산출` 버튼 확인.
7. 모의주문 게이트 확인.
