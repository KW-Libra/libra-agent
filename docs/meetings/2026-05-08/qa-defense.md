# 예상 질문 답변표

## Q1. 다이렉트 인덱싱이 뭔지 제목만 보고 모르겠다.

답변: 제목에서는 Direct Indexing을 앞세우지 않고, “개인 투자자를 위한 설명 가능한 AI 멀티 에이전트 기반 자동 포트폴리오 리밸런싱 시스템”으로 정의했습니다. Direct Indexing은 사용자가 직접 고른 종목으로 나만의 포트폴리오를 구성한다는 투자 방식이고, LIBRA는 그 포트폴리오를 관리하고 리밸런싱 판단을 돕는 시스템입니다.

## Q2. 그래서 정확히 뭘 만드는 건가?

답변: 사용자가 종목과 목표비중을 설정하면, 시스템이 현재 잔고와 시장 정보를 확인하고, AI 멀티 에이전트가 리밸런싱 여부를 판단한 뒤, 판단 근거와 후보 주문을 사용자에게 보여주는 웹 서비스입니다.

## Q3. 그냥 기계적 리밸런싱과 뭐가 다른가?

답변: 기계적 리밸런싱은 목표비중 이탈만 봅니다. LIBRA는 목표비중 이탈에 더해 공시, 뉴스, 리포트, 수익성, 거래비용 관점을 분리해 확인하고 Judge Agent가 종합 판단합니다. 또한 어떤 근거로 판단했는지를 Decision Trace로 남깁니다.

## Q4. 그냥 모듈러 시스템 아닌가? 왜 멀티 에이전트인가?

답변: 단순 모듈러 시스템은 정해진 순서대로 함수를 실행합니다. LIBRA는 Judge Agent가 현재 상황을 보고 어떤 관점 에이전트를 호출할지 선택하고, 호출하지 않은 이유도 기록합니다. 각 에이전트는 독립 관점의 의견을 내고, Judge가 충돌을 조정해 최종 판단합니다.

## Q5. 수익률이 좋아진다는 근거가 있나?

답변: 아직 수익률 우위를 단정하지 않습니다. 그래서 검증 지표를 Buy & Hold, 기계적 비중 리밸런싱, LIBRA 판단 기반 리밸런싱 세 가지로 나누어 비교하려고 합니다. 핵심 지표는 비용 차감 후 수익률, MDD, 거래횟수, 회전율입니다.

## Q6. 뉴스나 공시 같은 정성 데이터를 어떻게 정량화할 건가?

답변: 각 문서와 이벤트를 공통 신호 형식으로 변환합니다. 예를 들어 방향성, 강도, 신뢰도, 출처 신뢰도, 영향 기간, 위험 수준을 필드로 두고, Judge가 이 값을 기반으로 관점별 의견을 비교합니다. LLM 출력은 그대로 쓰지 않고 스키마 검증과 정규화를 거칩니다.

## Q7. 비용을 고려한다는 게 무슨 뜻인가?

답변: 후보 리밸런싱 초안이 생기면 거래규모, 매수·매도 방향, 수수료, 세금, 슬리피지, 스프레드 같은 실행 마찰을 비용 관점에서 검토합니다. 최종 결정은 Judge가 수익 관점과 비용 관점을 함께 보고 내립니다.

## Q8. 실전 자동매매까지 되는가?

답변: 현재 범위는 모의투자 주문 실행 경로와 사용자 승인 게이트입니다. 실전 자동매매는 승인 절차와 안전장치 검증이 더 필요하므로 현재는 의도적으로 제한했습니다.

## Q9. 현재 실제로 동작하는 것은 무엇인가?

답변: 회원 인증, 목표 포트폴리오 정의 저장, KIS 자격증명 등록, 잔고 동기화, 목표비중 대비 드리프트 계산, AI Judge 판단, Decision Trace 저장, KIS 현재가 기반 주문 수량 산출까지 연결되어 있습니다.

## Q10. 팀원별 기여는 어떻게 나눌 건가?

답변: 기능 컴포넌트 기준으로 나누되, 모든 팀원이 최소 하나의 에이전트 또는 에이전트 입력/출력 컴포넌트를 맡도록 조정합니다. Judge Agent는 종합 판단을 담당하고, 나머지 팀원은 뉴스, 리포트, 비용/수익, UI/리포팅과 연결된 관점 에이전트를 맡는 방식으로 정리하겠습니다.
