# 2026-05-08 지도교수 미팅 브리프

## 한 줄 목표

LIBRA는 개인 투자자가 직접 고른 종목과 목표비중을 기준으로, 시장 정보와 거래비용을 함께 고려해 리밸런싱 여부를 설명 가능하게 판단하고 모의주문까지 연결하는 AI 멀티 에이전트 기반 포트폴리오 리밸런싱 시스템이다.

## 내일 미팅에서 보여줘야 할 것

1. 프로젝트명과 문제정의가 교수님 피드백대로 다시 정리되었는가.
2. 우리가 만들 서비스의 요구사항이 번호로 명확하게 정의되었는가.
3. 기능 아키텍처가 5개 컴포넌트와 데이터 흐름으로 설명되는가.
4. 왜 멀티 에이전트인지, 단순 모듈러 시스템과 무엇이 다른지 답할 수 있는가.
5. 현재 구현된 end-to-end 흐름이 실제로 동작한다는 증거가 있는가.
6. 수익률 검증을 어떻게 할지 비교군과 지표가 준비되었는가.
7. 팀원별 기여가 컴포넌트 단위로 정리되어 있는가.

## 발표 순서

1. 제목 재정의
2. 배경 스토리라인
3. 문제 정의
4. 요구사항 명세
5. 전체 기능 아키텍처
6. 멀티 에이전트 판단 구조
7. 정성 데이터 정량화 방식
8. 현재 구현된 흐름
9. 데모 시나리오
10. 검증 계획
11. 팀 역할과 통합 계획
12. 다음 작업

## 말의 중심

- Direct Indexing은 제목의 주인공이 아니라 대상 투자 방식이다.
- LIBRA의 핵심은 단순 자동매매가 아니라 판단 근거가 남는 리밸런싱 의사결정이다.
- 멀티 에이전트는 여러 관점이 독립 의견을 내고 Judge가 조정한다는 점에서 의미가 있다.
- 수익률 우위는 주장하지 않고, Buy & Hold / 기계적 리밸런싱 / LIBRA 비교로 검증하겠다고 말한다.

## 현재 구현 증거

- 회원 인증, 포트폴리오 정의 저장, KIS 자격증명 등록, 잔고 동기화 경로가 존재한다.
- 백엔드가 `portfolio_definition`을 agent에 전달한다.
- agent가 목표비중과 현재비중의 드리프트를 계산해 후보 리밸런싱 초안을 만든다.
- Judge가 하위 에이전트 호출 결과를 decision trace로 남긴다.
- `REBALANCE` 판단 후 KIS 현재가 기반 모의주문 수량 제안이 생성된다.

## 오늘 검증한 스모크 결과

- 입력: 목표 포트폴리오 삼성전자 50%, SK하이닉스 50%, 현재 현금 100%
- Agent 결과: `REBALANCE`
- 후보 리밸런싱 초안: `005930 +0.1`, `000660 +0.1`
- 호출 에이전트: `disclosure`, `profit`, `cost`
- KIS 주문 제안: 2건 생성
- 예시: 삼성전자 `BUY 11주`, 현재가 `271,500원` 기준
